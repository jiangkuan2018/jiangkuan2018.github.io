toString方法

Posted on 2020-10-05 Edited on 2020-10-12

每个对象都有一个 toString() 方法，当该对象被表示为一个文本值时，或者一个对象以预期的字符串方式引用时自动调用。默认情况下，toString() 方法被每个 Object 对象继承。如果此方法在自定义对象中未被覆盖，toString() 返回 “[object type]”，其中 type 是对象的类型。MDN

object对象的toString方法返回[object Object]，当我们使delete String.prototype.toString删除字符串对象原型上的toString方法之后，调用字符串的toString方法会返回[object String]是因为字符串原型上重写了toSting方法，删除之后就会调用到 Object 对象中的toString方法，String是继承至Object对象的
每一个内置对象的原型都可以重写 toString方法，如果没有重写那么就会直接使用Object对象里的。使用 toString方法做对象检测就是直接调用Object.prototype.toString方法，使用call调用被检测的对象作为参数

现在我们知道了Object是js中所有类型的父类，而js中所有类型都有继承到Object.prototype.toString方法
那么我们现在来看一下js中各种类型是怎么重写toString方法的。

css类名的规则

Posted on 2020-07-01 Edited on 2020-09-28

前几天遇到一个 querySelectorAll 方法获取元素错误的问题，从错误信息来看.test$class是一个不合法的选择器。

https://developer.mozilla.org/zh-CN/docs/Web/API/Document/querySelectorAll
这里说明一下querySelectorAll方法接受一个字符串为参数，这个字符串必须是一个合法的 CSS selector ，如果不是会抛出一个 SyntaxError 错误

那么什么是不合法的选择器呢，查阅了一番文档发现只规定了哪些是正确的，并没有直接说明哪些字符不可以使用。

https://stackoverflow.com/questions/448981/which-characters-are-valid-in-css-class-names-selectors#comment14194934_448981
https://pineco.de/css-quick-tip-the-valid-characters-in-a-custom-css-selector/
https://www.w3.org/TR/CSS21/syndata.html#characters
贴几个搜索到的资料

stackoverflow回答里提到了一个正则css类名需要符合 /-?[_a-zA-Z]+[_a-zA-Z0-9-]*/
同样的 pineco 这个网站里也有提到正确的class 和 id的命名规则
w3.org则是官方写过的规定
所以大家在写css时尽量统一成BEM命名规范

document.referrer为空

Posted on 2020-06-18

记录一下今天解决的问题，sdk 的内置属性中有记录来源 url，但是客户今天反馈问题说跳转的页面来源 url 为空，仔细翻阅了一下资料发现 https 的站点跳转 http 的时候 http 站点不会记录document.referrer

记录一下站点没有document.referrer的情况

  1. 直接在浏览器地址栏中输入地址
  2. 从 https 的网站直接进入一个 http 协议的网站
  3. a标签设置rel="noreferrer"（兼容 IE7+）
  4. meta标签来控制不让浏览器发送referer
