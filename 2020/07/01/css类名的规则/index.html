css类名的规则

Posted on 2020-07-01 Edited on 2020-09-28

前几天遇到一个 querySelectorAll 方法获取元素错误的问题，从错误信息来看.test$class是一个不合法的选择器。

https://developer.mozilla.org/zh-CN/docs/Web/API/Document/querySelectorAll
这里说明一下querySelectorAll方法接受一个字符串为参数，这个字符串必须是一个合法的 CSS selector ，如果不是会抛出一个 SyntaxError 错误

那么什么是不合法的选择器呢，查阅了一番文档发现只规定了哪些是正确的，并没有直接说明哪些字符不可以使用。

https://stackoverflow.com/questions/448981/which-characters-are-valid-in-css-class-names-selectors#comment14194934_448981
https://pineco.de/css-quick-tip-the-valid-characters-in-a-custom-css-selector/
https://www.w3.org/TR/CSS21/syndata.html#characters
贴几个搜索到的资料

stackoverflow回答里提到了一个正则css类名需要符合 /-?[_a-zA-Z]+[_a-zA-Z0-9-]*/
同样的 pineco 这个网站里也有提到正确的class 和 id的命名规则
w3.org则是官方写过的规定
所以大家在写css时尽量统一成BEM命名规范
